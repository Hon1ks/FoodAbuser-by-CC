## Часть 1: Стоимость за 1 распознавание фото (платные API)

### Методология расчёта

Для корректного сравнения принята единица — **1 распознавание = 1 фото еды** с возвратом КБЖУ.

---

### 1.1 GPT-4o Vision

**Модель:** `gpt-4o` (или `gpt-4o-2024-08-06`)

**Расчёт стоимости:**
- Входной токен (текст): **$2.50 / 1M токенов**
- Выходной токен: **$10.00 / 1M токенов**
- Изображение `detail: high` (~512×512 px tile): **85 base tokens + 170 tokens/tile**
  - Типичное фото еды (1024×768): ~765 image tokens
  - Системный промпт + текст: ~300 tokens
  - Ответ JSON с КБЖУ: ~200 tokens

**Итого за 1 запрос:**
- Input: (765 image + 300 text) = 1065 tokens × $2.50/1M = **$0.00266**
- Output: 200 tokens × $10.00/1M = **$0.00200**
- **Итого: ~$0.0047 за 1 запрос** (округлённо ~$0.005)

> Примечание: при `detail: low` изображение стоит фиксированные 85 tokens (~$0.0002) — но качество распознавания падает. Не рекомендуется для еды.

| Объём | Стоимость |
|-------|-----------|
| 1 запрос | ~$0.005 |
| 100 запросов | ~$0.47 |
| 1 000 запросов | ~$4.70 |

### 2.1 Google Gemini Flash (бесплатный tier)

**Модель:** `gemini-1.5-flash` / `gemini-2.0-flash`
**API:** Google AI Studio / Vertex AI

#### Формат запроса
- **Base64** или **URL** (через `fileData` с предварительной загрузкой в Files API)
- Поддерживается inline base64 в `inlineData`

```json
POST https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=API_KEY
{
  "contents": [
    {
      "parts": [
        {
          "inlineData": {
            "mimeType": "image/jpeg",
            "data": "/9j/4AAQSkZJRgABAQAA..."
          }
        },
        {
          "text": "Identify all food items in this photo and return calories, protein, fat, carbs as JSON"
        }
      ]
    }
  ]
}
```

#### Лимиты бесплатного tier (Google AI Studio, 2025)
| Параметр | gemini-1.5-flash | gemini-2.0-flash |
|----------|------------------|------------------|
| RPM (запросов/мин) | 15 | 15 |
| RPD (запросов/день) | 1 500 | 1 500 |
| TPM (токенов/мин) | 1 000 000 | 1 000 000 |
| Платный tier | $0.075/1M input tokens | $0.10/1M input tokens |

> При платном использовании Gemini Flash ~в 25–30x дешевле GPT-4o по токенам.

#### Качество распознавания еды
- Близко к GPT-4o по пониманию контекста
- Хорошо распознаёт европейские, азиатские и русские блюда
- Умеет оценивать объём порции по визуальным подсказкам (тарелка, ложка)
- Benchmarks (VQA, FoodSeg103): Gemini Flash уступает GPT-4o примерно на 5–10% по точности нутриентов
- Недетерминирован, как и GPT-4o

#### КБЖУ vs название
- **Умеет возвращать КБЖУ** при правильном промпте (JSON с calories, protein, fat, carbs)
- Поддерживает `responseMimeType: "application/json"` для структурированного ответа

#### Сравнение с GPT-4o
| Критерий | Gemini Flash | GPT-4o |
|----------|-------------|--------|
| Точность распознавания | ~88–92% | ~93–96% |
| Оценка порции | Хорошая | Отличная |
| КБЖУ из фото | Да | Да |
| Русские блюда | Хорошо | Отлично |
| Бесплатный лимит | 1500 req/day | Нет |
| Цена (платный) | ~$0.0002/запрос | ~$0.005/запрос |

**Вердикт:** Отличная бесплатная альтернатива GPT-4o. При 1500 запросов/день достаточно для MVP и малого продакшена.

### 2.2 Groq + Llama 3.2 Vision

**Модели:** `llama-3.2-11b-vision-preview`, `llama-3.2-90b-vision-preview`
**API:** https://api.groq.com/openai/v1/chat/completions (OpenAI-совместимый)

#### Формат запроса
- **Base64** (через `data:image/jpeg;base64,...`) или **URL**
- OpenAI-совместимый формат messages

```json
POST https://api.groq.com/openai/v1/chat/completions
Authorization: Bearer GROQ_API_KEY

  "model": "llama-3.2-11b-vision-preview",

    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {
            "url": "data:image/jpeg;base64,/9j/4AAQ..."
          }
        },
        {
          "type": "text",
          "text": "Identify food and return КБЖУ as JSON"
        }
      ]
    }
  ]

```

#### Лимиты бесплатного tier (2025)
| Параметр | llama-3.2-11b-vision | llama-3.2-90b-vision |
|----------|---------------------|---------------------|
| RPM | 30 | 15 |
| RPD | 7 000 | 3 500 |
| TPM | 7 000 | 7 000 |
| Платный tier | $0.18/1M tokens | $0.74/1M tokens |

> **7 000 запросов/день бесплатно** — самый щедрый бесплатный лимит из всех vision API.

#### Качество распознавания еды
- Llama 3.2 11B Vision: умеренное качество, ~80–85% для стандартных блюд
- Llama 3.2 90B Vision: значительно лучше, ~88–92%, сопоставимо с Gemini Flash
- Известная проблема: сложные составные блюда и редкая кухня — хуже GPT-4o

#### КБЖУ vs название

- Качество КБЖУ-расчётов ниже, чем у GPT-4o: ошибка по калориям может достигать ±30%
- Рекомендуется комбинировать: Groq распознаёт блюдо → CalorieNinjas/Nutritionix выдаёт КБЖУ

#### Сравнение с GPT-4o
| Критерий | Groq 11B | Groq 90B | GPT-4o |
|----------|----------|----------|--------|
| Точность распознавания | ~82% | ~90% | ~95% |
| Скорость ответа | Очень быстро (<1с) | Быстро (~1–2с) | Умеренно (2–5с) |
| КБЖУ из фото | Частично | Да | Да |
| Бесплатный лимит | 7000/день | 3500/день | Нет |
| Цена (платный) | ~$0.0001/запрос | ~$0.0004/запрос | ~$0.005/запрос |

**Вердикт:** Groq — лучший вариант по соотношению бесплатного лимита и скорости. Для MVP с умеренными требованиями к точности — отличный выбор. 90B модель конкурентоспособна с Gemini Flash.

### 2.3 Ollama + LLaVA (локальная модель)

**Проект:** https://ollama.com
**Модели:** `llava:7b`, `llava:13b`, `llava:34b`, `llava-llama3`, `bakllava`

#### Формат запроса
- **Base64** (через локальный REST API)
- Нет интернет-зависимости

```bash
# Запуск модели
ollama pull llava:13b

# REST запрос
curl http://localhost:11434/api/generate -d '{
  "model": "llava:13b",
  "prompt": "What food is in this image? Estimate calories, protein, fat, carbs. Return JSON.",
  "images": ["/9j/4AAQSkZJRgABAQAA..."]
}'
```

#### "Лимиты" (самохостинг)
- **Нет лимитов по запросам** — ограничено только железом
- Стоимость: только электричество + сервер
- llava:7b требует: ~8 GB VRAM (GPU) или ~16 GB RAM (CPU-режим)
- llava:13b требует: ~16 GB VRAM или ~32 GB RAM
- На CPU: 5–30 секунд на запрос (зависит от железа)
- На GPU (RTX 3080): <2 секунды

#### Качество распознавания еды
- LLaVA 7B: ~75–80% распознавание еды — заметно хуже облачных моделей
- LLaVA 13B: ~82–85% — приемлемо для простых блюд
- LLaVA 34B: ~88% — близко к Gemini Flash, но требует серьёзного железа
- **Слабости:** сложные составные блюда, редкая кухня, точность оценки порции
- **Сильные стороны:** полная приватность, нет сетевых задержек, нет стоимости API

#### КБЖУ vs название
- Умеет выдавать КБЖУ при правильном промпте, но точность нутриентов низкая
- Рекомендуется: LLaVA распознаёт блюдо → передать текст в локальную БД (USDA Open Data)
- Для точного КБЖУ лучше интегрировать с CalorieNinjas или Nutritionix

#### Сравнение с GPT-4o
| Критерий | LLaVA 7B | LLaVA 13B | GPT-4o |
|----------|----------|-----------|--------|
| Точность распознавания | ~78% | ~84% | ~95% |
| КБЖУ из фото | Слабо | Частично | Да |
| Стоимость | $0 (железо) | $0 (железо) | $0.005/запрос |
| Приватность | Полная | Полная | Нет |
| Скорость (GPU) | 1–2с | 2–4с | 2–5с |
| Лимиты | Нет | Нет | Нет (платно) |

**Вердикт:** Подходит для self-hosted решений с требованиями к приватности. Для продакшена нужен GPU-сервер. Качество ниже облачных решений, но при нулевой операционной стоимости оправдано в определённых сценариях.

### 2.4 HuggingFace — food recognition модели

**Пример модели:** `nateraw/food` (fine-tuned ViT на Food-101 датасете)
**Inference API:** `https://api-inference.huggingface.co/models/nateraw/food`

#### Формат запроса
- **Binary (multipart)** или **base64 JSON**

```python
import requests

API_URL = "https://api-inference.huggingface.co/models/nateraw/food"
headers = {"Authorization": "Bearer hf_YOUR_TOKEN"}

with open("food.jpg", "rb") as f:
    data = f.read()

response = requests.post(API_URL, headers=headers, data=data)
# Ответ: [{"label": "pizza", "score": 0.987}, ...]
```

#### Лимиты бесплатного tier (HuggingFace Inference API)
| Параметр | Free (Community) | PRO ($9/мес) |
|----------|-----------------|--------------|
| Запросов/час | ~100–300 (нестабильно) | Приоритет |
| Cold start | Да (30–60с пауза) | Нет |
| Rate limit | Низкий, неофициальный | Выше |
| SLA | Нет | Нет |

> **Важно:** HuggingFace Inference API для бесплатных пользователей работает нестабильно — модели "засыпают" и требуют прогрева. Не подходит для продакшена без платного плана.

#### Другие food-модели на HuggingFace
| Модель | Классов | Датасет | Точность |
|--------|---------|---------|---------|
| `nateraw/food` | 101 | Food-101 | ~90% top-1 |
| `eslamxm/food-classifier` | 101 | Food-101 | ~88% |
| `Kaludi/food-category-classification-v2.0` | 12 категорий | Custom | ~95% (категории) |
| `julien-c/hotdog-not-hotdog` | 2 | Custom | — (демо) |

#### КБЖУ vs название
- **Только классификация** — возвращает название блюда из фиксированного набора классов
- Нет расчёта КБЖУ, нет оценки веса порции
- Необходима интеграция с Nutritionix / CalorieNinjas для КБЖУ
- Ограничен датасетом Food-101 (101 класс американских/европейских блюд)

#### Качество и ограничения
- Высокая точность для блюд из Food-101 (~88–92%)
- **Критический недостаток:** только 101 класс — борщ, пельмени, плов и т.д. отсутствуют
- Не подходит для русской кухни без дополнительного fine-tuning
- Не оценивает объём порции

#### Сравнение с GPT-4o
| Критерий | HF Food Model | GPT-4o |
|----------|--------------|--------|
| Точность (Food-101 блюда) | ~90% | ~97% |
| Русские/нестандартные блюда | 0% (нет в датасете) | Отлично |
| КБЖУ из фото | Нет | Да |
| Количество классов | 101 | Неограничено |
| Стоимость | Бесплатно (до лимита) | $0.005/запрос |

**Вердикт:** Подходит только для ограниченного набора блюд (западная кухня). Для FoodAbuser с русской аудиторией — **не рекомендуется** как основной метод.

### 2.5 Clarifai — реальные лимиты бесплатного tier

**Официальный free tier:** 1 000 операций/мес

#### Что считается "операцией"
- 1 вызов food-item-recognition model = **1 операция**
- Batch из 128 изображений = **128 операций**
- Нет разделения по типу модели — все операции одинаковы

#### Реальные ограничения
| Параметр | Free tier |
|----------|-----------|
| Операций/мес | 1 000 |
| Изображений в batch | до 128 |
| Rate limit | ~100 RPM |
| Коммерческое использование | Ограничено (ToS) |
| SLA | Нет |
| Срок действия free operations | Не переносятся на следующий месяц |

> **Реальный лимит для FoodAbuser:** 1000 распознаваний/мес = ~33 в день. Это мало для любого активного пользователя (допустим, 3 приёма пищи × 10 пользователей = 30/день уже на пределе).

#### Сравнение с заявленными лимитами
- Официально: 1 000/мес — **соответствует действительности**
- Нет скрытых лимитов по RPM в документации (на практике ~60–100 RPM)
- После исчерпания free tier — запросы возвращают ошибку 402, не деградируют
